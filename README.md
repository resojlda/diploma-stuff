# diploma-stuff
<p>Возможности</p>
<p>Система для поиска выпускников учебных заведений и анализа их трудоустройства.</p>
<p>Начальный функционал:</p>
<br>
1. Поиск и анализ:<br>
	1.1 По университету (сравнение с другими университетами)<br>
	1.1.1 Соотношение трудоустроившихся к нетрудоустроившимся<br>
	1.1.2 Временной промежуток до трудоустройства<br>
	1.1.3 Проходили ли интернатуру/стажировку<br>
	1.1.4 Временной промежуток до Junior<br>
	1.1.5 Временной промежуток до Middle<br>
	1.1.6 Временной промежуток до Senior<br>
1.2 По году выпуска<br>
	1.2.1. Временной промежуток до Junior<br>
	1.2.2 Временной промежуток до Middle<br>
	1.2.3 Временной промежуток до Senior<br>
	2.2.4 Выбор технологии<br>
	1.3 По месту работу<br>
		1.3.1 Где учились (университет, факультет, если необходимо) (в каком городе)<br>
		1.3.2 Количество выпускников, принятых на работу (с указанием по годам)<br>
		1.3.3 Используемые технологии (+частота смены места работы для каждой из них)<br>
		1.3.4 Количество молодых сотрудников (трудоустроившихся, не окончив ВУЗ)<br>
	1.4 По городу<br>
1.4.1 Соотношение работающих в городе обучения к тем, которые не работают в городе обучения<br>
1.4.2 Cоотношение работающих к неработающим<br>
</p><br>
<p>Обязанности</p>
<p>В соответствии с указанным функционалом, распределение обязанностей:</p>
1)Гобрач Денис<br>
Визуальная часть системы – работа с представлением данных. Сервис дает возможность выбирать визуальный вид ответа: будь то диаграммы, графики или текстовые данные. Также работа, связанная с упрощением работы пользователя, к примеру, выпадающие списки, подставляющие значения по мере печати среди имеющихся.<br>
2)Емельянов Сергей<br>
Первоначально – настройка работы сервера, и работы сайта: регистрация, аутентификация и авторизация, возможность делиться информацией в социальных сетях, возможность выбирать формы представления информации, смотреть историю – различные удобства, связанные с использованием сервиса. <br>
3)Марышева Александра<br>
Подключение и работа с базой данных для сохранения, кеширования и обновления информации. Работа системы подразумевает, что если запрос выполняется впервые, данные собираются и анализируются, записываются в бд для последующего быстрого извлечения. Данные, которые не используются, будут удалены, а данные, которые использовались чаще, будут обновлены, для сохранения целосности базы.<br>
4)Провоторов Вадим<br>
Работа с подключением к API, получением Access-token-a, необходимого для доступа к данным каждому пользователя персонально, настройка и упрощение API социальной сети непосредственно внутри проекта (оболочка, позволяющая слать запросы посредством вызова метода, когда нам не обязательно знать, что происходит внутри метода). Также работа над версткой и дизайном.<br>
5)Степаненко Александр<br>
Вызывая API методы, обрабатывать информацию и считать статистику, описанную выше. В связи с сложностями, вызванными ограничением социальной сети на количество ответов (одна тысяча за запрос) и запросов (три в секунду), необходимость в дополнительных мерах деления информации на порции, чтоб извлечь как можно больше самым быстрым способом. <br>
